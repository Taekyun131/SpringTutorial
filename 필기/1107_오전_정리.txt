정리질문
1. 클라이언트가 request를 할 때 공부해야 하는 것 3가지
-> Method    : get, post
    URL         : 요청
    Parameter : 서버에게 전송할 데이터

2. url의 구성하는 각 부분을 설명하시오.
-> protocol   : http(웹프로토콜)
    host      서버주소 (아이피 또는 도메인)
    port       80
    컨텍스트패스   :  프로젝트 단위 (WAS)
    value, query    :  스프링에게 요청하는 서비스

3. 스프링에서  @Controller의 의미
1. 요청을 받아준다. 이때 파라미터가 있다면 받아 준다. 
2. 비지니스 로직을 수행한다.. 비지니스 로직이란 데이터베이스 작업을
   의미한다.. 즉 비지니스 로직이 필요하면 서비스 레이어에 작업을
   시킨다.
3. 응답을 해 준다.
   동기방식에서는 view를 지정한다.
   비동기방식에서는 문자열, 또는 숫자, 객체로 전송한다.

4. 스프링에서 RequestMapping의 의미

  알려준다.. 
  스프링에서는 어노테이션으로 스프링에서 제공하는 기능을 부여한다.
